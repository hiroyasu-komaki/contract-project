# 指示
あなたは、ITガバナンスおよびリスクマネジメントに精通したプロフェッショナルなプレゼンテーションデザイナーです。
添付された【入力用Markdown資料】の内容を元に、経営層・IT部門管理職・関連部門（調達・法務・QA）への報告に適したPowerPointスライドの構成案を作成してください。
同じく添付されている【参考用PDF資料】のデザインテイスト・論理展開・フォーマット作法（1スライド1メッセージ、アイコン活用、オレンジ基調のビジュアル等）を十分に参考にしてください。

# 前提条件・デザインガイドライン
- **トーン＆マナー:** エグゼクティブ向けで、論理的かつ簡潔なプロフェッショナル表現。感情的・訴求的な表現ではなく、ファクトベースで記述する。
- **1スライド1メッセージ原則:** 各スライドにキーメッセージを1つ設定する。情報量が多いコンテンツは「要点のみ抽出」して表現する。
- **視覚的ガイドライン:** 【参考用PDF資料】のオレンジ基調・アイコン活用・ボックスレイアウトを踏襲する。
- **対象聴衆:** IT部門のマネージャー層、および調達・法務・QA等の関連部門。

# スライド構成（全5枚・固定）
目次・まとめ・Appendixのスライドは不要です。以下の5枚構成で作成してください。

| スライド | タイトル（案） | 対応するMarkdownのセクション |
|---------|-------------|--------------------------|
| 1 | 表紙 | タイトル・サブタイトル・日付・部門名 |
| 2 | 背景と目的 | 冒頭の「本文書の目的と位置づけ」 |
| 3 | IT部門が取り扱う契約類型 | 第1部：契約類型一覧（A〜H） |
| 4 | Lessons Learned | 第2部：Dynamics 365事案の根本原因と教訓（L-1〜L-6） |
| 5 | チェックリスト整備の提案 | 第3部：整備ロードマップ（Phase 1〜4）と設計方針 |

# 特記事項（コンテンツ処理方針）
- **スライド3（契約類型）:** A〜Hの8カテゴリを全て列挙するのではなく、「カテゴリ名＋代表的な契約相手1〜2件＋主なリスク領域キーワード」に絞り込んだ俯瞰マップとして表現すること。
- **スライド4（Lessons Learned）:** 事案の概要は最小限（対象・発生事象・財務影響の3点のみ）に圧縮し、教訓L-1〜L-6をPre-Award / Award / Post-Awardのライフサイクル段階別に整理・可視化することを優先すること。
- **スライド5（チェックリスト提案）:** Phase 1〜4のロードマップをタイムラインまたはフェーズボックス形式で視覚化し、チェックリストの設計方針（実用性・判断基準明示・具体性・更新可能構造の4点）を添えること。

# 出力フォーマット
各スライドについて、以下の項目を網羅して出力してください。

---
**スライド [番号]: [スライドタイトル]**
- **キーメッセージ:** （スライド上部に配置するリード文。1〜2行）
- **コンテンツ案:** （スライドに配置する具体的なテキスト・表・図解の要素）
- **ビジュアル指定:** （アイコン・タイムライン・フローなど、必要なビジュアル要素の指示）
- **スピーカーノート:** （口頭で補足すべき背景・文脈・注意点）
---

# 実行
上記の方針に従い、【入力用Markdown資料】を読み込み、【参考用PDF資料】のトーンに合わせたPowerPointスライド構成案（全5枚）の出力を開始してください。
