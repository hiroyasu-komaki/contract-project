以下の仕様で新しいSkillsとPythonスクリプトを作成してください。

## 作成するもの

### 1. Pythonスクリプト（`src/`）

`src/main.py` を作成してください。以下の2つのサブコマンドを持つCLIツールです。

```
python src/main.py summarize <入力ファイルパス>
python src/main.py review
```

**`summarize` サブコマンド：**
- `input/<指定ファイル>` を読み込む
- 拡張子は `.doc` または `.docx` に対応する
- `.docx` は `python-docx` ライブラリで読み込む
- `.doc` は `antiword` コマンド（システムにインストール済みの場合）またはテキスト変換にフォールバックする
- テキストを抽出してMarkdownファイルに出力する
- 出力先：`mid/contract_YYYYMMDD_HHmmss.md`
- `mid/` ディレクトリが存在しない場合は自動作成する

**`review` サブコマンド：**
- `mid/contract_*.md` の中から最新ファイルを自動的に選択して読み込む
- レビュー結果のテンプレート（後述）をMarkdown形式で出力する
- 出力先：`output/legalreview_YYYYMMDD_HHmmss.md`
- `output/` ディレクトリが存在しない場合は自動作成する

レビュー内容の生成ロジックはClaude APIは使わず、テンプレート出力のみ実装してください（分析はClaude Codeが担当します）。

出力テンプレートは以下の構造にしてください：

```markdown
# 法務レビュー：{契約書ファイル名}
レビュー日時：{YYYY年MM月DD日 HH:MM}
対象ファイル：{mid/contract_*.md のパス}

## 1. 条文別 有利不利分析

| 条文 | 内容概要 | 甲への影響 | 乙への影響 | 備考 |
|------|---------|-----------|-----------|------|
| （Claude Codeが記入） | | | | |

## 2. 総合的な有利不利バランス

（Claude Codeが記入）

## 3. 該当する可能性のある法規制・業界規制

（Claude Codeが記入）

---
*本レビューは参考情報です。法的判断が必要な事項については、別途専門家への相談をお勧めします。*
```

### 2. 依存パッケージの追加

`requirements.txt` に以下を追記してください：

```
python-docx
```

追記後、以下を実行してください：

```bash
source .venv/bin/activate
pip install python-docx
```

### 3. Skillsファイル（`.claude/commands/`）

**`.claude/commands/contract-summarize.md`：**

```
引数で受け取ったファイル名（$ARGUMENTS）の契約書を読み込んで要約してください。

1. 以下のコマンドを実行する：
   `.venv/bin/python src/main.py summarize "$ARGUMENTS"`

2. 生成された `mid/contract_*.md` の最新ファイルを読む。

3. 以下の構成で日本語の要約を作成し、そのファイルに追記する：
   - **契約の種類・目的**：何のための契約か
   - **当事者**：甲・乙それぞれの立場と役割
   - **契約期間・有効期限**
   - **条文一覧と概要**：各条文のタイトルと1〜2文の説明
   - **特記事項**：特に注意が必要な条項（罰則・免責・自動更新など）
```

**`.claude/commands/legalreview.md`：**

```
mid/ ディレクトリの最新の契約書要約ファイルを読んで、法務視点でのレビューを作成してください。

1. 以下のコマンドを実行する：
   `.venv/bin/python src/main.py review`

2. 生成された `output/legalreview_*.md` の最新ファイルを読む。

3. 以下の2点についてレビュー結果を記入する：

   **【条文別 有利不利分析】**
   - 各条文について、甲・乙それぞれへの影響を分析する
   - 有利／不利／中立 で評価し、その理由を簡潔に説明する
   - 表の最後に総合的な有利不利バランスをまとめる
   - 一方的に不利な条項がある場合は特に明示する

   **【該当する可能性のある法規制・業界規制】**
   - 契約内容に関連する法律・規制を列挙するにとどめる（例：個人情報保護法、不正競争防止法、独占禁止法など）
   - 各規制について1〜2文で関連性を説明する
   - 詳細な法的判断が必要な場合は「追加調査が必要です。別途専門家へのご相談をお勧めします。」と記載する

4. ファイル末尾の免責事項は削除しないこと。
```

### 4. フォルダ構成の更新

以下のディレクトリを追加してください（既存の場合はスキップ）：

```bash
mkdir -p input mid src
```
